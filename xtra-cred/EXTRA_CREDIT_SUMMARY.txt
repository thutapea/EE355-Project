OVERVIEW
--------
A content-based friend recommendation system has been implemented for the
TROJANBOOK network application. This system intelligently recommends potential
friend connections based on user similarity metrics derived from their profile
attributes and characteristics.

KEY FEATURES IMPLEMENTED
------------------------

1. METADATA SUPPORT FOR USERS
   - Extended Person class with a map<string, string> for storing user metadata
   - New methods:
     * void addAttribute(string key, string value) - Add/update metadata
     * string getAttribute(string key) - Retrieve metadata value
     * map<string, string> getMetadata() - Get all metadata

2. SIMILARITY SCORING ALGORITHM
   - Weighted scoring system evaluating multiple compatibility factors:
     
     Factor                Weight    Scoring Logic
     -----------------------------------------------------------
     Age Difference        25%       Closer ages get higher scores
     Email Domain          20%       Same domain = perfect match
     Phone Area Code       15%       Matching area codes preferred
     College               15%       Same college = high compatibility
     Field of Study        15%       Same field = strong match
     Zip Code              10%       Same location = bonus points
     
   - Total score normalized to 0-100 percentage

3. RECOMMENDATION ENGINE
   - Method: Network::recommendFriends(string fname, string lname, int topN)
   - Features:
     * Calculates similarity to all non-friend users
     * Sorts by compatibility score (highest first)
     * Returns top N recommendations with scores
     * Intelligently excludes:
       - The user themselves
       - Already-connected friends

4. USER INTERFACE
   - New menu option 8: "Friend Recommendations"
   - Interactive flow:
     1. User selects option 8
     2. Enter target person's name
     3. System displays top 5 recommendations with compatibility scores
     4. Clean formatted output showing ranking and percentage match

TECHNICAL IMPLEMENTATION
------------------------

Files Modified/Created:
  - person.h: Added metadata map and methods
  - person.cpp: Implemented metadata management
  - network.h: Added recommendation methods and made search public
  - network.cpp: Implemented similarity calculation and recommendations
  - test_recommendations.cpp: Created test harness with sample data setup
  - recommendTest.txt: Database with 11 test users

Algorithm Details:
  1. Metadata is stored as key-value pairs in each Person object
  2. For recommendations, iterate through all users in the network
  3. Calculate similarity score for each non-friend user
  4. Apply bubble sort to rank by score (descending order)
  5. Display top N results with compatibility percentages

Complexity Analysis:
  - Time: O(n²) where n = number of users (compare each pair)
  - Space: O(n) for storing scores and metadata
  - Suitable for networks up to several thousand users
