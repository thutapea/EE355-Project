OVERVIEW
--------
The Extra Credit Friend Recommendation System has been successfully implemented
for the TROJANBOOK network application. This system intelligently recommends
potential friend connections based on user similarity metrics derived from their
profile attributes and characteristics.

SYSTEM FEATURES IMPLEMENTED
---------------------------

1. METADATA SUPPORT FOR USERS
   - Extended Person class with a map<string, string> for storing user metadata
   - New methods:
     * void addAttribute(string key, string value) - Add/update metadata
     * string getAttribute(string key) - Retrieve metadata value
     * map<string, string> getMetadata() - Get all metadata
   - Allows storing: college, field of study, zip code, and other attributes

2. SIMILARITY SCORING ALGORITHM
   - Weighted scoring system evaluating multiple compatibility factors:
     
     Factor                Weight    Scoring Logic
     -----------------------------------------------------------
     Age Difference        25%       Closer ages get higher scores
     Email Domain          20%       Same domain = perfect match
     Phone Area Code       15%       Matching area codes preferred
     College               15%       Same college = high compatibility
     Field of Study        15%       Same field = strong match
     Zip Code              10%       Same location = bonus points
     
   - Total score normalized to 0-100 percentage
   - Intelligent weighting reflects real-world connection patterns

3. RECOMMENDATION ENGINE
   - Method: Network::recommendFriends(string fname, string lname, int topN)
   - Features:
     * Calculates similarity to all non-friend users
     * Sorts by compatibility score (highest first)
     * Returns top N recommendations with scores
     * Intelligently excludes:
       - The user themselves
       - Already-connected friends
   - Time complexity: O(n²) where n = number of users
   - Space complexity: O(n)

4. USER INTERFACE INTEGRATION
   - New menu option 8: "Friend Recommendations"
   - Interactive flow:
     1. User selects option 8 from main menu
     2. Enter target person's first and last name
     3. System displays top 5 recommendations with compatibility scores
     4. Clean formatted output showing ranking and percentage match
     5. Returns to main menu for additional actions

Algorithm Flow:
  1. Receive user name and retrieve Person object
  2. For each other user in network:
     - Skip if same person or already friends
     - Calculate similarity score (0-100)
     - Store score with person pointer
  3. Sort all scores in descending order using bubble sort
  4. Display top N results with names and compatibility percentages

Scoring Function Logic:
  - Age: Calculate year difference, award points based on proximity
  - Email: Extract domain, award full points if match
  - Phone: Extract area code (first 3 digits), award if match
  - College: Direct string comparison from metadata
  - Field: Direct string comparison from metadata
  - Zip: Direct string comparison from metadata
  - Final score = (total points / max points) * 100
